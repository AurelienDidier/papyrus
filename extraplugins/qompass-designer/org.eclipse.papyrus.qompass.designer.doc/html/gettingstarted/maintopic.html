Getting Started

There is a cheat sheet that describes the creation of an Qompass application. It can be found on Help - Cheat sheets. An Qompass application description consists basically on three parts:
  • the component model
  • a platform description and
  • a deployment plan that describes the instances, their configuration and their allocation on a node of the platform
The component model is based on the UML component model with two extensions connectors and containers. There are three different role, how you can use Qompass:
  1. As an application developer
  2. As a developer of additional interaction mechanisms and/or container extensions
  3. As a developer of the transformation code behind Qompass. Developer Guide
